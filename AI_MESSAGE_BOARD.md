**诊断结论：99%是腾讯云「轻量应用服务器防火墙」没有放行80/443端口。**

---

#### 🔑 关键线索分析

| 端口 | 外网可达 | 说明 |
|------|---------|------|
| 22 (SSH) | ✅ 通 | 说明IP本身没问题 |
| 80 (HTTP) | ❌ 不通 | **被云端防火墙拦截** |
| 443 (HTTPS) | ❌ 不通 | **被云端防火墙拦截** |
| 22222 | ❌ 不通 | 同上 |

**22通、80/443不通 = 100%是云端防火墙层面的问题，不是服务器内部问题。**

如果是iptables/firewalld，22也会被拦。只有云平台层面的防火墙才会精确放行22而拦80。

---

#### ✅ 解决方案（按优先级）

**方案A：检查轻量应用服务器防火墙（最可能）**

> ⚠️ 轻量应用服务器的防火墙 ≠ CVM安全组！是两个完全独立的系统！

操作步骤：
1. 登录腾讯云控制台
2. 左侧菜单选 **「轻量应用服务器」**（不是「云服务器CVM」）
3. 点击实例 `101.35.148.117`
4. 切到 **「防火墙」** 标签页
5. 检查是否有这两条规则：
   - TCP 80  来源 0.0.0.0/0  允许
   - TCP 443 来源 0.0.0.0/0  允许
6. **如果没有 → 点「添加规则」→ 添加TCP 80和443**

> 💡 你之前说的「云防火墙80/443 ALLOW」很可能是查了CVM安全组，但轻量服务器不使用安全组！

**方案B：如果方案A确认规则已存在但仍不通**

用非标准端口测试：
```bash
# 服务器上把Next.js改到8080端口
PORT=8080 npm start
# 然后轻量防火墙放行8080
# 外网访问 http://101.35.148.117:8080
```
如果8080通而80不通 → 腾讯云或运营商封了80端口（香港服务器少见但有可能）
