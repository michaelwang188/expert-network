## 四、P2 级修复（优化项）

### 4.1 首页按钮风格统一

| 位置 | 当前 | 建议 |
|------|------|------|
| 首页Hero | `注册加入` / `登录账户` | `免费注册` / `登录` |
| 导航栏 | `退出` | `退出登录` |

**理由**: `注册加入` 是3个字+2个字冗长，`免费注册` 更吸引人。`登录账户` 相比 `登录` 多了2个冗余字。

### 4.2 emoji 使用规范

| 场景 | 当前 | 建议 |
|------|------|------|
| 首页特性展示 | 🔬🧠🛡️💰🎓 | 保持不变 |
| Dashboard 空状态 | ✅ | 保持不变 |
| Toast 成功消息 | 无 emoji | 加 ✅ 前缀 |
| 管理后台侧边栏 | 全部有 emoji | 保持不变 |
| 按钮 | 无 emoji | **不加**（保持简洁） |
| 表单标签 | 无 emoji | **不加** |

**原则**: emoji 用于视觉装饰和情绪表达，不用于功能按钮和表单。

### 4.3 Toast 消息规范

**当前问题**: 订单操作Toast仅有简单动词（`已接单`、`标记完成`），缺乏上下文。

**规范**:

| 操作 | 当前Toast | 规范Toast |
|------|---------|----------|
| 专家接单 | `已接单` | `✅ 已接单，请及时联系研究员确认访谈时间` |
| 标记完成 | `标记完成` | `✅ 已标记完成，等待管理员确认结算` |
| 确认结算 | `已结算` | `✅ 结算完成，积分已发放` |
| 取消订单 | `已取消` | `订单已取消` |
| 保存成功 | `✅ 保存成功！...` | 保持现状 ✅ |
| 操作失败 | `操作失败，请重试` | `操作失败，请稍后重试` |

**Toast 结构**: `{emoji} {结果动词}，{后续引导}`
