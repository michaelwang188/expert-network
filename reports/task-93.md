### 3.3 合规手册
```
## 合规手册

**文档版本**: V2.0  
**发布日期**: 2026年6月1日  
**适用对象**: 所有用户（研究员、专家、管理员）

---

### 一、数据保护

**数据收集原则**
• 最小必要原则：只收集必要的信息
• 告知同意原则：收集前告知用户，获得同意
• 目的限定原则：只能用于告知的目的

**数据存储原则**
• 境内存储：数据存储在境内服务器
• 加密存储：敏感数据加密存储
• 访问控制：最小权限原则

**数据删除原则**
• 用户删除账户后，个人数据应在30天内删除或匿名化
• 订单记录、咨询记录因合规要求，可匿名化保留

---

### 二、信息披露

**可以披露的信息**
• 公开信息（财报、新闻、行业报告）
• 个人观点（明确标注"个人观点"）
• 行业趋势（基于公开数据的分析）

**禁止披露的信息**
• 客户商业秘密（未公开的产品、战略、数据）
• 内幕信息（未公开的并购、融资、业绩）
• 他人隐私（同事、客户、第三方的私密信息）

**披露审批流程**
• 专家在咨询中提及未公开信息 → 立即报告合规团队
• 合规团队评估 → 决定是否继续或终止
• 如违反，按《违规处理办法》处罚

---

### 三、利益冲突

**常见利益冲突**
• 专家是客户竞争对手的员工
• 专家持有客户股票/衍生品
• 专家与客户有业务往来

**申报流程**
1. 发现冲突 → 立即申报（/compliance/conflict）
2. 合规团队评估 → 决定是否继续
3. 如继续 → 签署保密协议+利益冲突声明
4. 如拒绝 → 平台补偿（¥200咨询费）

**未申报后果**
• 首次发现：警告+扣分20
• 再次发现：降级+罚款¥1000
• 造成严重后果：封号+追责
```

**数据收集**

**数据泄露应急响应**
1. 发现泄露 → 立即报告合规团队
2. 评估影响 → 24小时内
3. 通知用户 → 72小时内
4. 通知监管机构 → 按法律要求
5. 修复漏洞 → 立即
