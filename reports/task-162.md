## 🎯 一、设计背景与目标

### 1.1 为什么要设计行业分类体系？

产研通ProLink作为专家对接平台，需要一套科学、完整、可扩展的行业分类体系，以实现：

1. **专家标签化**：为每位专家打上行业标签，便于搜索和匹配
2. **需求分类**：企业发布需求时，选择相关行业，精准推荐专家
3. **数据分析**：按行业统计专家分布、需求热度、费率水平等
4. **报告生成**：生成分行业的趋势报告、专家洞察报告

### 1.2 设计目标

| 目标 | 说明 |
|------|------|
| **科学性** | 对标国际通用标准（GICS），确保专业性 |
| **本土化** | 结合中国国情，参考申万分类，符合国内习惯 |
| **完整性** | 覆盖主流行业，同时支持新兴行业（如AI、元宇宙） |
| **可扩展性** | 支持三级、四级分类，适应未来发展 |
| **易用性** | 专家和企业都能快速理解和使用 |

### 1.3 设计原则

1. **互斥性**：每个专家/企业只能归属到一个最相关的二级行业
2. **完备性**：所有专家/企业都能找到对应的行业
3. **稳定性**：行业分类不宜频繁变动（每年评审一次）
4. **前瞻性**：预留新兴行业的分类空间

## 🔍 二、对标分析：GICS vs 申万

### 2.1 GICS（全球行业分类标准）

**简介**：由MSCI和标普开发，全球最常用的行业分类标准。

**结构**：11个一级行业 → 24个二级行业 → 69个三级行业 → 158个四级行业

| 一级行业（英文） | 一级行业（中文） |
|-----------------|-----------------|
| Information Technology | 信息技术 |
| Health Care | 医疗健康 |
| Financials | 金融 |
| Consumer Discretionary | 非必需消费品 |
| Consumer Staples | 必需消费品 |
| Industrials | 工业 |
| Energy | 能源 |
| Utilities | 公用事业 |
| Materials | 材料 |
| Real Estate | 房地产 |
| Communication Services | 通信服务 |

**优点**：
- ✅ 全球通用，国际认可度高
- ✅ 层级清晰，分类科学
- ✅ 定期更新，适应市场变化

**缺点**：
- ❌ 偏重上市公司，对未上市企业覆盖不足
- ❌ 部分行业分类不符合中国国情（如房地产单独分类）
- ❌ 新兴行业（AI、新能源）分类不够细致

### 2.2 申万行业分类

**简介**：申万宏源研究所开发，国内最常用的行业分类标准。

**结构**：31个一级行业 → 134个二级行业 → 346个三级行业

| 一级行业 | 代表 |
|----------|------|
| 计算机 | 软件开发、IT服务 |
| 电子 | 半导体、消费电子 |
| 医药生物 | 化学制药、生物制品 |
| 电力设备 | 新能源、光伏设备 |
| 机械设备 | 通用设备、专用设备 |

**优点**：
- ✅ 符合中国国情，A股上市公司全覆盖
- ✅ 分类细致，三级行业达346个
- ✅ 国内机构广泛使用

**缺点**：
- ❌ 主要服务投资研究，对产业专家分类不够友好
- ❌ 部分行业划分过细（如"装饰建材"单独分类）
- ❌ 新兴行业更新不及时

### 2.3 对标结论

**产研通行业分类体系**应结合GICS的科学性和申万的本土化，同时针对**专家对接场景**进行优化。

## 🏗️ 三、产研通行业分类体系（ProLink Taxonomy）

### 3.1 体系结构

```
一级行业（20个）
  └── 二级行业（100+个）
      └── 三级行业（300+个，可选）
```

### 3.2 与GICS/申万映射

| ProLink一级 | GICS一级 | 申万一级 | 说明 |
|-------------|----------|----------|------|
| 信息技术 | Information Technology | 计算机、电子 | 合并计算机和电子 |
| 医疗健康 | Health Care | 医药生物 | 增加"医疗服务" |
| 金融 | Financials | 银行、非银金融 | 保持 |
| 消费品 | Consumer Staples + Discretionary | 食品饮料、家电等 | 合并必需和非必需 |
| 工业 | Industrials | 机械设备、汽车等 | 保持 |
| 能源 & 新材料 | Energy + Materials | 石油石化、有色等 | 合并能源和新材料 |
| 房地产 | Real Estate | 房地产 | 保持 |
| 通信 & 传媒 | Communication Services | 传媒、通信 | 合并 |
| 公用事业 | Utilities | 公用事业 | 保持 |
| **新能源** | ❌（分散在多个行业） | 电力设备（部分） | **新增，单独分类** |
| **人工智能** | ❌（分散在多个行业） | 计算机（部分） | **新增，单独分类** |
| **半导体** | ❌（属于信息技术） | 电子（部分） | **新增，单独分类** |

**说明**：产研通新增了"新能源"、"人工智能"、"半导体"三个一级行业，因为这些是当前的**热门领域**，专家需求量大，需要单独分类。

## 📖 六、行业覆盖范围说明

### 6.1 信息技术 vs 人工智能 vs 半导体

**问题**：这三个行业有重叠，如何划分？

**解答**：

| 场景 | 归属行业 | 说明 |
|------|----------|------|
| 开发AI算法的专家 | 人工智能 | 核心能力是AI算法 |
| 开发AI芯片的专家 | 半导体 | 核心能力是芯片设计 |
| 开发AI应用的专家 | 信息技术 | 核心能力是软件开发 |
| 既懂AI又懂芯片的专家 | 人工智能（优先） | 按最核心能力归属 |

**原则**：每位专家最多选择2个行业标签（主标签+辅标签）。

### 6.2 新能源 vs 汽车

**问题**：新能源汽车归属哪个行业？

**解答**：
- 新能源汽车**整车企业** → 汽车
- 新能源汽车**电池/电机/电控** → 新能源
- 充电桩运营 → 新能源

### 6.3 跨行业专家

**问题**：有些专家的研究领域跨越多个行业（如"AI+医疗"），如何分类？

**解答**：
- 选择**最核心**的1-2个行业
- 或者归属到"跨行业综合"
- 在个人简介中说明"AI+医疗"跨界背景

## 🔗 七、专家与行业匹配规则

### 7.1 专家注册时选择行业

**流程**：
1. 专家选择**一级行业**（必选，最多2个）
2. 专家选择**二级行业**（必选，最多3个）
3. 专家填写**行业经验年限**
4. 专家上传**行业相关成果**（论文、项目、专利）

### 7.2 审核规则

| 审核项 | 规则 |
|--------|------|
| 学历背景 | 与所选行业相关（如计算机专业→信息技术） |
| 工作经历 | 在相关行业工作≥3年 |
| 成果证明 | 有相关论文/专利/项目 |
| 推荐人 | 有同行业专家推荐（加分项） |

### 7.3 企业搜索专家时的匹配逻辑

**权重**：
- 一级行业匹配：40%
- 二级行业匹配：30%
- 关键词匹配：20%
- 地理位置：10%

**示例**：
企业搜索"半导体芯片设计专家" → 优先推荐：
1. 一级行业=半导体，二级行业=芯片设计
2. 个人简介中含"芯片设计"关键词
3. 上海地区（如果企业选择"上海"）

## 🚀 八、实施建议

### 8.1 数据库设计

```sql
-- 行业分类表
CREATE TABLE industry (
  id INT PRIMARY KEY,
  level INT NOT NULL, -- 1=一级, 2=二级, 3=三级
  parent_id INT, -- 父级行业ID
  name_zh VARCHAR(50) NOT NULL, -- 中文名称
  name_en VARCHAR(50), -- 英文名称
  description TEXT, -- 覆盖范围说明
  created_at TIMESTAMP
);

-- 专家-行业关联表
CREATE TABLE expert_industry (
  expert_id INT,
  industry_id INT,
  is_primary BOOLEAN, -- 是否主行业
  experience_years INT, -- 行业经验年限
  PRIMARY KEY (expert_id, industry_id)
);
```

### 8.2 专家注册流程改造

1. **新增行业选择步骤**：在注册流程中增加"选择行业"步骤
2. **智能推荐行业**：根据专家履历，自动推荐行业标签
3. **审核环节**：平台审核专家的行业选择是否合理

### 8.3 企业需求发布流程改造

1. **新增行业选择**：企业发布需求时，必须选择相关行业
2. **智能推荐专家**：根据行业标签，自动推荐匹配专家
3. **行业热度展示**：展示各行业的专家数量、需求热度

### 8.4 数据迁移

**现有专家的行业标签化**：
- 步骤1：根据专家履历，批量打上行业标签
- 步骤2：专家登录后，确认/修改行业标签
- 步骤3：平台审核，确保准确性
