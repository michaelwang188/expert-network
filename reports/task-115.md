## 🔴 当前实现缺陷

```ts
// src/app/api/requests/route.ts:71
const found = SENSITIVE.filter(w => 
  (outline || "").includes(w) || (title || "").includes(w)
)
```

| 缺陷 | 说明 |
|------|------|
| 仅字面匹配 | 全角/空格/符号插入全绕过 |
| 无 Unicode 标准化 | NFKC/NFD 未做 |
| 无前端预检 | 前端 `request/page.tsx` 依赖后端返回结果，不自己做 |
| 无拼音检测 | 74词×拼音变体可全绕过 |

## 修复建议

1. **Unicode 标准化**: `text.normalize('NFKC')` 去全角/变体
2. **去空格+符号**: `text.replace(/[\s·•.。，,、]/g,'')` 
3. **零宽字符去除**: `text.replace(/[\u200B-\u200D\uFEFF]/g,'')`
4. **正则变体匹配**: 词间允许 0-2 个任意字符 `并.?购`

> 拼音检测成本高可暂缓，先修字符标准化。
