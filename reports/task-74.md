## 动态验证

提交含「毒丸计划」（仅在后端词库中存在）的提纲：

```
POST /api/requests {"title":"正常调研","outline":"讨论毒丸计划的实施效果分析"}
→ HTTP 400: "提纲含敏感词，请修改后重新提交"
→ sensitiveWords: ["毒丸计划"]
```

✅ 后端防御有效 — 前端漏掉的词仍被后端拦截。

## 其他发现

1. 后端注释声明「共74个」，实际75个（42+33，注释误写32个歧义词）。
2. `page.tsx` 注释同样声明「74词」，但数组实际75个 — 与后端一致。
3. 合规中心页注释写「与后端 requests/route.ts 对齐（74词）」— 数量声明错误，但内容正确对齐。
