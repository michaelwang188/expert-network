三、发现的问题和缺失功能

🔴 P0 — 阻塞级（影响核心流程）
1. 测试账号登录问题：seed 脚本中 prisma/seed.ts 的 upsert 使用了 update: {}，导致生产数据库中测试账号密码未更新。已修复代码（commit b4a253af），需重新运行 npx prisma db seed。
2. 专家库页面无登录保护：/experts 页面是客户端组件，未调用 getServerSession()，未登录用户也能访问。应添加 session 检查或至少显示登录提示。
🟠 P1 — 重要级（影响用户体验）
3. 赛道分布数据硬编码：dashboard/page.tsx 第 38-40 行，6 条赛道的计数是写死的 mock 数据，未从数据库实时统计。应改为：prisma.request.groupBy({ by: ['industry'], _count: true })
4. 需求状态流转断裂：需求从 COMPLIANCE_OK 无法自动流转到 MATCHING，缺少管理员复核后触发状态变更的 API。
5. 无 IM / 通讯功能：研究员与专家之间没有消息系统，只能通过订单状态间接交互。
6. 无访谈预约：interviewTime 字段存在但没有任何 UI 或 API 可以设置它。
🟡 P2 — 改善级（优化体验）
7. 无中间件统一鉴权：每个页面各自检查 session（页面级），没有 middleware.ts 统一拦截。建议添加 matcher 规则统一处理。
8. 无支付接口：订单有 PAID 状态但无任何支付网关集成，目前是纯状态标记。
9. 无评价系统：专家有 rating 字段但无评价流程和 UI。
10. 无分页：专家列表 API 不支持 offset/limit 分页，大量数据时有性能风险。
11. 无密码强度校验：注册页面接受任意长度密码，无复杂度要求。
12. 导航栏不统一：某些页面用内联 style，某些依赖 Tailwind，编码风格不统一。

🟡 Schema 建议
1. ComplianceLog.handlerId 应添加外键关联到 User（目前只是裸 String）
2. Expert.rateHour 类型为 Int（单位：分），注册 API 中默认值设为 600000（6000元/小时），数值较大，建议统一为"元"或"角"
3. 缺少 Message 模型用于 IM 通讯
4. 缺少 Review 模型用于专家评价
5. 缺少 Notification 模型用于系统通知

6.2 冒烟测试 checklist

  1. 访问 https://expert-network-sooty.vercel.app → 应跳转到 /login
  2. 注册研究员账号 → 自动登录 → 跳转到 /dashboard
  3. 浏览 /experts → 看到专家列表（如无数据，先由管理员审核至少 1 位专家）
  4. 点击某个专家 → 进入详情页，查看信息完整
  5. 访问 /request → 填写调研需求 → 提交 → 看到成功提示
  6. 访问 /orders → 看到刚创建的订单（状态：待确认）
  7. 尝试取消该订单 → 状态变为「已取消」→ 成功
  8. 切换到管理员账号 → /admin → 看到待审核专家和待派单
  9. 审核通过一位专家 → 派送一个订单 → 确认开始

  11. 管理员访问 /compliance → 测试敏感词检测工具
